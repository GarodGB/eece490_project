# AI Advisor — Final Grounded Design

The chatbot is designed as a grounded academic advisor, not a generic chatbot.

## What it uses

The advisor reads the logged-in student's:

- major and current GPA;
- completed courses, credits, grades, and semesters;
- failed courses and weak passes;
- area strengths and weaknesses;
- unlocked/prerequisite status;
- recommendations generated by the ML-centered recommender;
- target-GPA feasibility outputs.

## How it answers

The advisor answers using project data first:

- GPA feasibility: compares target GPA with the ML-estimated plan and the mathematical GPA cap.
- Course recommendations: summarizes the same recommendations shown on the dashboard.
- Strength/weakness analysis: computes area averages from completed courses.
- Retake advice: distinguishes true failed courses from weak passes.
- Course-specific questions: checks catalogue status, prerequisites, and predicted difficulty.

If `OPENAI_API_KEY` is configured, the advisor can use an LLM fallback. The LLM prompt is strictly grounded in app data and is instructed not to invent course facts or university policies.

## Why this matters for Option A

This improves the application layer without replacing the ML work. The chatbot explains the outputs of the five-model recommendation system in student-friendly language, while keeping rules and ML roles clear:

- Rules validate eligibility.
- ML predicts difficulty, success probability, expected grade, workload, and risk.
- The advisor explains the result and flags unrealistic GPA goals.
