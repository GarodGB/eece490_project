# AcademicPath ML Models — Final Version

This project uses five supervised scikit-learn models. Runtime prediction helpers live in `services/ml_service.py`. Training scripts live in `ml/`, and trained artifacts are saved in `ml/models/`.

The models are trained on synthetic student-course histories generated by `scripts/generate_synthetic_data.py`. The course catalogue and prerequisite structure are real/planning-based, but student outcomes are synthetic because real student grades were not available.

---

## Model 1 — Personalized Course Difficulty

| Item | Description |
|---|---|
| File | `ml/model1_course_difficulty.py` |
| Task | Regression |
| Target | `1 - grade_points / 4`, using pre-attempt features only |
| Output | Difficulty score and category: Easy / Medium / Hard |
| Runtime use | Completed-course difficulty, available-course difficulty, recommendation ranking |

Important features include course level, credits, prerequisite count/depth, lab status, course pressure, prior GPA, recent average, prerequisite average, completed credits, term load, workload tolerance, and prerequisite completion ratio.

---

## Model 2 — Semester Workload Estimation

| Item | Description |
|---|---|
| File | `ml/model2_semester_workload.py` |
| Task | Regression |
| Target | Semester workload index from observed difficulty, credits, labs, and course mix |
| Output | Predicted semester difficulty and overload risk |
| Runtime use | Recommendation summary, semester planner, ML evidence section |

Model 2 estimates the whole semester, not a single course. It helps the system avoid recommending an overloaded schedule when a student wants GPA protection.

---

## Model 3 — Future Academic Risk

| Item | Description |
|---|---|
| File | `ml/model3_academic_risk.py` |
| Task | Classification |
| Target | Future academic risk in later semesters |
| Output | Low / Medium / High / Critical risk |
| Runtime use | Recommendation caution, dashboard risk signal, advisor explanations |

The model uses past academic performance and workload/difficulty context to estimate future risk. It should be discussed as a risk-support tool, not a deterministic judgment about a student.

---

## Model 4 — Course Success Probability

| Item | Description |
|---|---|
| File | `ml/model4_course_success_probability.py` |
| Task | Classification |
| Target | Whether the student earns C+ or above |
| Output | Course success probability |
| Runtime use | Recommendation fit score and course-level explanations |

Key features include prior GPA, recent average, prerequisite average, subject average, area strength, workload tolerance, course level, credits, prerequisite completion, term load, weak/strong area flags, and course role.

---

## Model 5 — Expected AUB Grade Points

| Item | Description |
|---|---|
| File | `ml/model5_expected_grade.py` |
| Task | Regression |
| Target | Expected course grade points out of 4.3 |
| Output | Expected course grade points |
| Runtime use | GPA-target planning and course ranking |

This model is essential because pass probability alone cannot answer whether a course helps the student raise GPA. Individual course grades can be 4.3 for A+, but cumulative GPA is capped at 4.0.

---

## Correct interpretation

AcademicPath is a hybrid academic planning system:

```text
Rules = eligibility and safety filter
ML = personalized prediction and ranking
UI = student-friendly explanation and warnings
```

Rules prevent invalid advice, such as recommending already-passed courses or courses without prerequisites. ML ranks valid candidates by expected difficulty, expected grade, success probability, workload, and risk.

---

## How to train all models

```powershell
.\.venv\Scripts\python.exe ml\train_all_models.py --regenerate
```

This regenerates the synthetic training data and trains all five models.

## What to say in the final defense

Good claim:

> The project uses deterministic academic rules for validity and supervised ML models for personalized ranking, expected grade, difficulty, workload, and academic risk.

Avoid this claim:

> The model is proven accurate for real AUB students.

Real student outcomes were not available, so the strongest honest claim is that this is a rigorous ML-assisted prototype.
